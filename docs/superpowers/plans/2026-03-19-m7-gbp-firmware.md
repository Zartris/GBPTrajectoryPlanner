# M7: GBP Firmware (ESP32-C5) — Implementation Plan

> **For agentic workers:** REQUIRED: Use superpowers:subagent-driven-development (if subagents available) or superpowers:executing-plans to implement this plan. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** A single ESP32-C5 running `RobotAgent` + `PhysicsState` over Embassy async tasks, receiving velocity commands from GBP, publishing `RobotBroadcast` over ESP-NOW, and reading position from encoder.

**Architecture:** Three Embassy tasks: `physics::physics_task` drives the wheel motor via PWM and integrates encoder pulses into `position_s`; `gbp::gbp_task` calls `RobotAgent::step()` each tick; `vis::vis_task` serialises `RobotStateMsg` over UART to the bridge. The map is baked into flash at build time as a postcard-encoded binary. All coordination state lives in `mk_static!` statics (no heap). ESP-NOW carries `RobotBroadcast` between robots using the split `EspNowSender` / `EspNowReceiver` API.

**Tech Stack:** Embassy async (`embassy-executor`, `embassy-time`), `esp-hal` + `esp-radio` from `branch = "main"`, `postcard`, `defmt`, `heapless`, `gbp-agent` (no_std, no alloc).

**Prerequisite:** M6 plan complete and passing.

// --- GBP CommsInterface over ESP-NOW ---

use gbp_comms::{CommsInterface, RobotBroadcast};

/// ESP-NOW backed CommsInterface.
/// Uses a staging buffer pattern: `broadcast()` serialises into `pending_buf`,
/// then the device's `gbp_task` performs the actual async send via `EspNowSender`.
pub struct EspNowComms {
    pub rx_buf:      heapless::Deque<RobotBroadcast, 8>,
    pub pending_buf: [u8; 250],
    pub pending_len: usize,
}

impl EspNowComms {
    pub fn new() -> Self {
        Self { rx_buf: heapless::Deque::new(), pending_buf: [0u8; 250], pending_len: 0 }
    }

    /// Alias for `new()` — used in host tests for clarity.
    pub fn new_test() -> Self { Self::new() }

    /// Returns the pending serialised bytes if a broadcast is queued, or `None`.
    pub fn pending_bytes(&self) -> Option<&[u8]> {
        if self.pending_len > 0 { Some(&self.pending_buf[..self.pending_len]) } else { None }
    }

    /// Clears the pending broadcast buffer after the async send completes.
    pub fn clear_pending(&mut self) { self.pending_len = 0; }
}

impl CommsInterface for EspNowComms {
    type Error = ();

    /// Serialise `msg` into `pending_buf`. The actual ESP-NOW send is async;
    /// `gbp_task` calls `pending_bytes()` + `sender.send_async()` + `clear_pending()`.
    fn broadcast(&mut self, msg: &RobotBroadcast) -> Result<(), Self::Error> {
        let bytes = postcard::to_slice(msg, &mut self.pending_buf).map_err(|_| ())?;
        self.pending_len = bytes.len();
        Ok(())
    }

    fn receive_broadcasts(&mut self) -> heapless::Vec<RobotBroadcast, 8> {
        let mut out = heapless::Vec::new();
        while let Some(msg) = self.rx_buf.pop_front() {
            let _ = out.push(msg);
        }
        out
    }
}
```

**Note on async send:** `EspNowComms::broadcast()` only serialises into `pending_buf`. The `gbp_task` (device-only, see Task 4) does the actual async send: calls `pending_bytes()`, awaits `sender.send_async(...)`, then calls `clear_pending()`.

// firmware/src/bins/esp32c5/src/physics/mod.rs

pub const PULSES_PER_METRE: f32 = 1000.0; // encoder resolution
pub const MAX_VELOCITY:     f32 = 1.5;    // m/s motor speed limit

/// Position/velocity state for one robot on a single edge.
#[derive(Clone, Copy)]
pub struct PhysicsState {
    pub position_s:  f32,
    pub velocity:    f32,
    pub edge_length: f32,
}

/// Integrate encoder pulses into position_s and velocity.
/// `pulses` = encoder ticks since last call; `dt_s` = elapsed seconds; `ppm` = pulses per metre.
pub fn integrate_encoder(state: &mut PhysicsState, pulses: u32, dt_s: f32, ppm: f32) {
    let dist = pulses as f32 / ppm;
    state.position_s += dist;
    state.velocity    = dist / dt_s;
}

/// Clamp motor velocity command to [0.0, MAX_VELOCITY].
pub fn clamp_velocity(v: f32) -> f32 {
    v.clamp(0.0, MAX_VELOCITY)
}

/// Embassy async task — runs at 100 Hz.
/// `velocity_cmd` is shared with `gbp_task` via AtomicU32 (f32 bits).
#[embassy_executor::task]
pub async fn physics_task(
    state:        &'static core::cell::Cell<PhysicsState>,
    velocity_cmd: &'static core::sync::atomic::AtomicU32,
) {
    use embassy_time::{Duration, Ticker};
    let mut ticker = Ticker::every(Duration::from_millis(10)); // 100 Hz

    loop {
        ticker.next().await;
        let dt_s: f32 = 0.01;

        // Read encoder pulses (placeholder — wire to HAL in hardware bring-up)
        let pulses: u32 = 0; // TODO: read from encoder peripheral

        let mut s = state.get();
        integrate_encoder(&mut s, pulses, dt_s, PULSES_PER_METRE);

        // Apply velocity command from GBP task
        let cmd = f32::from_bits(velocity_cmd.load(core::sync::atomic::Ordering::Relaxed));
        s.velocity = clamp_velocity(cmd);
        // TODO: write clamped velocity to motor PWM register

        state.set(s);
    }
}

// firmware/src/bins/esp32c5/src/gbp/mod.rs

use core::sync::atomic::{AtomicU32, Ordering};
use gbp_agent::{RobotAgent, StepObs};
use crate::comms::EspNowComms;
use crate::physics::PhysicsState;
// BROADCAST_ADDRESS is device-only (esp_radio not available in host test env)
#[cfg(not(test))]
use esp_radio::esp_now::BROADCAST_ADDRESS;

/// Embassy async task — GBP step at 20 Hz.
/// Device-only: Embassy tasks cannot be spawned in the host test harness.
/// The GBP logic itself (RobotAgent::step) is tested directly in the tests block below.
#[cfg(not(test))]
#[embassy_executor::task]
pub async fn gbp_task(
    agent:        &'static core::cell::RefCell<RobotAgent>,
    comms:        &'static core::cell::RefCell<EspNowComms>,
    sender:       &'static core::cell::RefCell<esp_radio::esp_now::EspNowSender<'static>>,
    phys_state:   &'static core::cell::Cell<PhysicsState>,
    velocity_cmd: &'static AtomicU32,
) {
    use embassy_time::{Duration, Ticker};
    let mut ticker = Ticker::every(Duration::from_millis(50)); // 20 Hz

    loop {
        ticker.next().await;

        let phys = phys_state.get();
        let obs = StepObs {
            position_s: phys.position_s,
            velocity:   phys.velocity,
            edge_id:    agent.borrow().current_edge(),
        };

        // GBP step — may call comms.broadcast() which serialises into comms.pending_buf
        let out = agent.borrow_mut().step(obs, &mut *comms.borrow_mut());
        velocity_cmd.store(out.velocity.to_bits(), Ordering::Relaxed);

        // Send the broadcast over ESP-NOW (async, device only)
        #[cfg(not(test))]
        {
            let mut comms_ref = comms.borrow_mut();
            if let Some(bytes) = comms_ref.pending_bytes() {
                let _ = sender.borrow_mut()
                    .send_async(&BROADCAST_ADDRESS, bytes).await;
                comms_ref.clear_pending();
            }
        }
    }
}

#[cfg(test)]
mod tests {
    use super::*;
    use crate::comms::EspNowComms;

    #[test]
    fn gbp_output_velocity_is_non_negative() {
        let map: gbp_map::Map = postcard::from_bytes(
            include_bytes!("../../../../../maps/test_loop.bin")
        ).unwrap();
        let edge = map.edges.first().unwrap();
        let mut comms = EspNowComms::new_test();
        let mut agent = gbp_agent::RobotAgent::new(0, &map, edge.id, 0.0);
        let obs = StepObs { position_s: 0.1, velocity: 0.2, edge_id: edge.id };
        let out = agent.step(obs, &mut comms);
        assert!(out.velocity >= 0.0, "velocity must be non-negative");
    }
}
```

**Note on async broadcast:** `EspNowComms::broadcast()` serialises into a `pending_buf: [u8; 250]` field and sets `pending_len`. The async send is done in `gbp_task` via `sender.send_async()`. Add `pending_buf`, `pending_len`, `pending_bytes()`, and `clear_pending()` to `EspNowComms` in `comms/mod.rs`.
